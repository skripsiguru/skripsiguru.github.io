9. jelaskan kelebihan pemrograman Java dibandingkan pemrograman assembler ?​


Pemrograman Java memiliki beberapa kelebihan dibandingkan pemrograman assembler, antara lain:

1. Portabilitas - Java dapat dijalankan di berbagai platform karena menggunakan platform-independent virtual machine (JVM), sedangkan assembler hanya bisa dijalankan pada platform tertentu.

2. Keamanan - Java memiliki fitur-built in security seperti sandboxing dan bytecode verification, sehingga dapat mencegah eksekusi kode yang berbahaya. Assembler tidak memiliki fitur keamanan ini.

3. Mudah dipelajari - Syntax Java lebih mudah dipelajari dibandingkan dengan assembler yang menggunakan kode mesin yang lebih kompleks.

4. Mudah dikembangkan - Dalam pengembangan perangkat lunak, penggunaan library dan framework pada Java lebih mudah dilakukan daripada dalam pengembangan menggunakan assembler yang lebih terbatas.

5. Pengembangan aplikasi yang lebih efisien - Dalam Bahasa pemrograman Java, pengembangan aplikasi berjalan lebih cepat karena program yang ditulis dalam bahasa assembler cenderung lebih kompleks dan memakan waktu yang lebih lama untuk dibangun.

Jawaban:

Java dan assembler adalah dua bahasa pemrograman yang sangat berbeda yang melayani tujuan yang berbeda. Java adalah bahasa pemrograman tingkat tinggi, sedangkan assembler adalah bahasa pemrograman tingkat rendah.

Berikut adalah beberapa keuntungan menggunakan Java daripada assembler:

Kemudahan penggunaan

: Java pada umumnya lebih mudah dipelajari dan digunakan daripada assembler karena memiliki sintaks yang lebih intuitif dan mengabstraksikan banyak detail tingkat rendah.

Portabilitas

: Kode Java dapat dijalankan di platform apa pun yang memiliki Java Virtual Machine (JVM) terinstal, tanpa perlu rekompilasi. Kode assembler, di sisi lain, spesifik untuk arsitektur yang ditulisnya dan perlu ditulis ulang atau dikompilasi ulang untuk dijalankan pada arsitektur yang berbeda.

Pemrograman berorientasi objek

: Java adalah bahasa pemrograman berorientasi objek, yang memungkinkan pembuatan kode modular dan dapat digunakan kembali. Assembler tidak memiliki dukungan bawaan untuk pemrograman berorientasi objek.

Manajemen memori

: Java memiliki manajemen memori otomatis melalui pengumpul sampahnya, yang membebaskan memori yang tidak lagi digunakan oleh program. Dalam assembler, manajemen memori harus dilakukan secara manual oleh programmer.

Penanganan kesalahan

: Java memiliki penanganan kesalahan bawaan melalui mekanisme pengecualiannya, yang memungkinkan penanganan kesalahan secara anggun saat runtime. Assembler tidak memiliki dukungan bawaan untuk penanganan kesalahan.

Pengertian Pemograman Java

Java adalah bahasa pemrograman yang multi platform dan multi device. Sekali anda menuliskan sebuah program dengan menggunakan Java, anda dapat menjalankannya hampir di semua komputer dan perangkat lain yang support Java, dengan sedikit perubahan atau tanpa perubahan sama sekali dalam kodenya. Aplikasi dengan berbasis Java ini dikompulasikan ke dalam p-code dan bisa dijalankan dengan Java Virtual Machine. Fungsionalitas dari Java ini dapat berjalan dengan platform sistem operasi yang berbeda karena sifatnya yang umum dan non-spesifik.

Kelebihan Dan Kekurangan Java

1. Kelebihan Java

2. Kekurangan Java

Kekurangan Java adalah penggunaan memori yang cukup banyak, lebih besar daripada bahasa tingkat tinggi sebelum generasi Java. Namun hal ini memang sesuai dengan fitur beragam yang dimiliki oleh Java. Masalah memori ini juga tidak dialami oleh semua pengguna aplikasi Java. Mereka yang sudah menggunakan perangkat keras dengan teknologi terbaru tidak merasakan kelambatan dan konsumsi memori Java yang tinggi. Lain halnya dengan mereka yang menggunakan teknologi lama atau komputer yang sudah berumur tua lebih dari empat tahun akan merasakan adanya kelambatan. Namun apapun kelemahan yang dimiliki Java, faktanya adalah Java merupakan bahasa pemrograman yang populer dan digunakan di seluruh dunia saat ini.

32. jelaskan penulisan struktur pemrograman java


Struktur dalam bahasa pemrograman Java merupakan sebuah format standar yang dikeluarkan oleh developer bagi para programmer. Struktur tersebut berbentuk serpti dibawah ini

package details

class className

{

Data members;

user_defined method;

public static void main (String args[])

{

Block of Statements

}

}

Penjelasan tiap struktur dalam bahasa pemrograman java adalah sebagai berikut

- Package. Merupakan kumpulan dari class, interace dan sub-packages. Sub package terdiri dari kumpulan class, interface dan sub-sub packages etc.java.lang.*; Package diimpor secara default dan disebut dengan default package

 

- Class. Merupakan keyword yang digunakan untuk mengembangkan tipe data user defined. Setiap program java harus dimulai dengan konsep class

- "ClassName" merupakan nama variabel valid dalam java yang dianggap sebagai nama dari setiap class. Setiap nama class dalam pemrograman Java dianggap sebagau tipe data user-defined.

- Data member. Melambangkan instance maupun static yang akan dipilih berdasarkan dari nama class

- User-defined. Melambangkan instance maupun static yang akan digunakan untuk melakukan operasi baik satu kali maupun setiap kali.

- Setiap program Java memulai eksekusi dari main method. Oleh sebab itu, main method disebut juga sebagai program driver.

- Karena sifat main method yang tidak mengembalikan nilai, maka return type harus dihindari.

- Karena sifat main method yang hanya dijalankan sekali sepanjang program, maka main method harus static.

- Karena sifat main method yang harus dapat diakses oleh setiap programmer java, maka access nya harus public.

- Setiap main method dalam java harus memakai array String

- Block of Statement. Melambangkan barisan program yang dapat dieksekusi yang memanggil user-defined method yang memiliki business-logic.

- Penamaan dalam pemrograman Java adalah pada setiap class yang mengandung main method, maka nama file harus sama dengan nama class dengan ditambahkan ekstensi .java

37. Jelaskan peranan java (JDK) di dalam pemrograman android​


Jawaban:

Mengembangkan bahasa java dalam android

Penjelasan:

Terdapat 2 jenis kompenen utama dalam java, yaitu:

- JRE (Java Runtime Envoironment)

- JDK (Java Developer Kit)

Java Runtime Envoironment

JRE merupakan java yang berfungsi untuk menjalankan program java, tidak ada unsur penyusunan disini. Proses menjalankan program java dilakukan oleh JVM (Java Virtual Machine).

Java Developer Kit

JDK merupakan java yang berfungsi untuk melakukan penulisan serta pengembangan program java. Dalam JDK terdapat JRE dan library-library java lainnya. JDK hanya digunakan oleh developer yang bekerja dengan java. Apabila JDK sudah terpasang, tidak diperlukan lagi pemasangan JRE.

NB: Android dapat dikembangkan dengan Kotlin, Java, dan C++. Ketiga bahasa ini berkerja sebagai back-end aplikasi yang terbentuk nantinya.
