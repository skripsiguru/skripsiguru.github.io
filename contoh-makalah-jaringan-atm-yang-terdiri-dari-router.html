16. Pada komunikasi yang terjadi diantara router, routing protokol yang mengijinkan router untuk berbagi (sharing) informasi tentang jaringan dan koneksi antar-router dalam jaringan disebut ....


Pada komunikasi yang terjadi diantara router, routing protokol yang mengijinkan router untuk berbagi (sharing) informasi tentang jaringan dan koneksi antar router dalam jaringan disebut ROUTING DINAMIS

Penjelasan:

Routing adalah proses pengiriman data atau informasi dengan meneruskan paket data dari satu jaringan ke jaringan lainnya.Beberapa contoh proses routing yaitu pengiriman pesan dari satu komputer ke komputer lain, telepon melalui jaringan internet dan pengeriman data ke jaringan komputer.Berdasarkan jenis konfigurasinya, routing dibedakan menjadi beberapa jenis, yaitu :Routing Statis yaitu jenis routing yang dilakukan oleh admin atau  pengelola jaringan untuk mengkonfigurasi informasi tentang jaringan yang dituju secara manual, caranya adalah dengan memasukan pengaturan routing ke dalam routing table dari router.Routing Dinamis, adalah jenis routing yang dilakukan oleh router dengan cara membuat jalur komunikasi data secara otomatis. Routing dinamis merupakan routing protokol yang digunakan untuk menemukan jaringan dan untuk melakukan update routing table pada router. Protokol pada Routing Dinamis dibedakan menjadi beberapa jenis, yaitu :Routing Information ProtocolInterior Gateway Routing ProtocolBorder Gateway Protocol

25. bagaimana konfigurasi jaringan vlan pada router​


Pengertian Inter VLAN Routing

Inter VLAN Routing adalah merupakan sebuah proses untuk melakukan forward traffic jaringan, dari sebuah VLAN menuju VLAN yang lain menggunakan router agar bisa saling berkomunikasi.

Ketika beberapa PC terhubung ke Switch dengan mode VLAN, maka antar PC tersebut tidak dapat berkomunikasi. Nah, disini saya akan menggunakan Router agar antar VLAN dapat saling berkomunikasi, hanya menggunakan satu interface saja. Oleh karena itu silahkan sobat perhatikan dan coba praktikan.

Topologi VLAN dan InterVLAN

Berikut adalah gambar topologi sederhana yang akan kita bahas pada kesempatan kali ini. Diharapkan setelah memahami konfigurasi ini semoga sobat bisa menambah VLAN lebih banyak lagi. Yang terpenting adalah memahami Cara Konfigurasi Inter VLAN pada Router.

< topologi vlan dan inter vlan routing

Berikut adalah tabel konfigurasi IP, subnet dan VLAN.

tabel konfigurasi ip

Tabel konfigurasi IP

Kofigurasi VLAN pada Switch

Langkah pertama dalah melakukan konfigurasi VLAN pada Switch sebagai berikut :

Klik Switch masuk ke Tab CLI dan ketikan kode di bawah ini

Penamaan VLAN

Switch>enable

Switch#conf t

Switch(config)#vlan 30

Switch(config-vlan)#name group.30

Switch(config-vlan)#exit

Switch(config)#vlan 40

Switch(config-vlan)#name group.40

Switch(config-vlan)#exit

Pengarahan port pada VLAN

Switch(config)#interface fa0/1

Switch(config-if)#switchport access vlan 30

Switch(config-if)#exit

Switch(config)#interface fa0/2

Switch(config-if)#switchport access vlan 40

Switch(config-if)#exit

Switch(config)#interface fa0/3

Switch(config-if)#switchport mode trunk

Switch(config-if)#exit

Konfigurasi Inter VLAN pada Router

Selanjutnya kita akan konfigurasi Router untuk bisa melakukan inter VLAN terhadap 2 VLAN yang berbeda pada Switch, agar PC0 dan PC1 dapat terhubung.

Klik pada perangkat Router kemudian pilih Tab CLI ketik no untuk mengawali, ketikan kode berikut ini :

Membuat interface Virtual dalam Interface Fisik

Router>enable

Router#conf t

Router(config)#interface gi0/0

Router(config-if)#no shutdown

Router(config-if)#exit

Router(config)#interface gi0/0.30

Router(config-subif)#encapsulation dot1Q 30

Router(config-subif)#ip address 192.168.30.1 255.255.255.0

Router(config-subif)#exit

Router(config)#interface gi0/0.40

Router(config-subif)#encapsulation dot1Q 40

Router(config-subif)#ip address 192.168.40.1 255.255.255.0

Router(config-subif)#exit

Bila sobat arahkan mouse di atas Router maka akan terlihat hasil Encapsulation IP pada GigabitEthernet0/0 terdapat 2 virtual IP.

40. Bagaimana cara mengirimkan data dalam konektivitas jaringan dengan menggunakan router? ​


Jawaban:

Untuk mengirim data melalui konektivitas jaringan menggunakan router, Anda dapat mengikuti langkah-langkah berikut:

1. **Siapkan Perangkat:**

- Pastikan Anda memiliki perangkat yang terhubung ke jaringan, seperti komputer atau perangkat seluler.

2. **Hubungkan ke Router:**

- Pastikan perangkat Anda terhubung ke router Anda melalui kabel Ethernet atau jaringan nirkabel (Wi-Fi).

3. **Alamat IP:**

- Perangkat Anda akan diberi alamat IP oleh router. Anda bisa memeriksa alamat IP perangkat dengan membuka pengaturan jaringan di perangkat Anda.

4. **Kirim Data:**

- Anda dapat mengirim data melalui jaringan dengan menggunakan berbagai aplikasi atau layanan, seperti email, peramban web (untuk mengakses situs web), aplikasi pesan instan, atau aplikasi lain yang memerlukan akses ke internet.

5. **Protokol Jaringan:**

- Data yang dikirimkan melalui jaringan akan mengikuti protokol jaringan tertentu, seperti TCP/IP (Transmission Control Protocol/Internet Protocol), yang digunakan untuk mengatur bagaimana data dikirim, diterima, dan diatur di jaringan.

6. **Routing:**

- Router adalah perangkat yang mengatur lalu lintas data di jaringan. Itu akan mengarahkan data yang dikirim dari perangkat Anda ke tujuan yang tepat dalam jaringan, baik itu perangkat lain di rumah Anda atau melalui internet ke tujuan di luar jaringan lokal Anda.

7. **Keamanan:**

- Pastikan Anda memiliki pengaturan keamanan yang tepat di router Anda, seperti kata sandi Wi-Fi yang kuat, firewall, dan pengaturan keamanan lainnya untuk melindungi jaringan Anda dari ancaman keamanan.

8. **Monitoring Jaringan:**

- Anda dapat menggunakan antarmuka administratif router (biasanya diakses melalui peramban web dengan memasukkan alamat IP router) untuk memantau aktivitas jaringan, mengatur pengaturan jaringan, dan mengelola perangkat yang terhubung.

Penting untuk memahami bahwa pengaturan router dapat bervariasi tergantung pada merek dan modelnya. Jika Anda memiliki pertanyaan khusus tentang pengaturan atau masalah jaringan dengan router Anda, lebih baik merujuk pada manual pengguna atau menghubungi dukungan teknis produsen router.
